1. Một câu giới thiệu (elevator pitch)
Đây là ứng dụng QA / kiểm tra báo cáo kết cấu (PDF): người dùng upload PDF, backend lưu file, rồi gọi Google Gemini để phân tích theo prompt + tiêu chí (criteria) lấy từ PostgreSQL. Frontend hiển thị kết quả upload và kết quả AI.

4. Backend: endpoint và trách nhiệm
POST /api/upload: đọc file, validate (kích thước, extension, magic bytes %PDF), lưu Document (binary trong DB, có ttl_delete_date).
GET /api/documents/{id}/download: stream PDF từ DB.
POST /api/analyze: load document + settings mặc định (base_prompt, criteria JSON), gọi get_ai_service() → hiện tại chỉ gemini, factory sẵn sàng mở rộng provider khác.
/health: health check cho load balancer / Docker.
Placeholder (chưa làm đầy đủ): job status, results by job, CRUD settings qua API (trong code ghi Phase 4–6).

5. Data model (để nói về “thiết kế dài hạn”)
Trong db_models.py có:

Settings: base_prompt + criteria (JSON) — đúng hướng cấu hình QA không hard-code trong code.
Document: PDF trong LargeBinary, TTL retention.
AnalysisJob, AnalysisResult: schema cho xử lý bất đồng bộ / lưu kết quả lâu dài — phù hợp roadmap job queue dù hiện analyze đang đồng bộ trong request.
6. Frontend: demo UX
Route: /upload → /result, / redirect upload (App.jsx).
Upload.jsx: drag-drop, validate PDF client-side (10MB), gọi uploadPDF rồi analyzeDocument, lưu sessionStorage, navigate.
api.js: base URL từ env, timeout mặc định 30s — lưu ý phỏng vấn: analyze lâu có thể cần tăng timeout hoặc chuyển sang job + polling (đúng với placeholder /api/jobs/...).
7. Cách chạy demo nhanh
Docker: docker-compose.yml — Postgres, backend (port nội bộ, healthcheck), frontend 8080:80, biến GEMINI_API_KEY bắt buộc.
Local dev: Backend README ghi Node 22 + Poetry; cần DATABASE_URL, GEMINI_API_KEY, chạy migrations Alembic theo docs trong Backend/docs/.
8. Góc “câu hình phỏng vấn” bạn có thể chủ động
Tại sao lưu PDF trong DB? Đơn giản hóa triển khai, backup theo transaction; trade-off: kích thước DB, có thể chuyển object storage sau.
Mở rộng AI provider? Đã có AIService abstraction + ai_factory; thêm OpenAI/Claude theo pattern tương tự Gemini.
Scalability: Analyze đồng bộ có giới hạn; model có AnalysisJob — có thể nói hướng worker + queue (Celery/RQ/Arq) và dùng endpoint status đã placeholder.
Bảo mật: PDF chứa dữ liệu nhạy cảm — HTTPS (docs/SSL-SETUP.md), CORS ALLOWED_ORIGINS, không expose backend ra ngoài trực tiếp trong compose (ghi chú nginx reverse proxy).
